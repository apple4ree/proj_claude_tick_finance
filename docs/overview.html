통합 진입점: /experiment

/experiment가 canonical. 기존 /iterate, /new-strategy는 deprecated. 한 줄 명령으로 Phase 1(α 발견) → Phase 2(agent chain 설계) → 백테스트 → 4-gate validate → BH benchmark → programmatic/agent feedback → iter_finalize → meta-review까지 자동.

/experiment --market crypto_1h --symbols BTCUSDT,ETHUSDT,SOLUSDT \
            --is-start 2025-07-01 --is-end 2025-10-31 \
            --oos-start 2025-11-01 --oos-end 2025-12-31 \
            --design-mode agent --feedback-mode both \
            --n-iterations 10     # minimum 10 enforced (policy 2026-04-19)
정책 (2026-04-19): --n-iterations는 반드시 ≥ 10. 1-shot 실행은 rejected (--smoke-test 플래그로만 bypass). 이유: 자율 루프가 실제 lesson을 축적하려면 최소 10 cycles 필요.

Agent Handoff Schema (Pydantic Layer)

3개 handoff 경계에 Pydantic 검증 강제 — "인용만 하고 계산은 안 하는 추측성 산출물" 방지.

Schema경계핵심 필수 필드 / 규칙
AlphaHandoffalpha-designer → execution-designer BriefRealismCheck: adjusted_ev_bps = brief_ev * horizon_scale − spread_cross − regime_adj (±max(0.5, 5%) 허용). decision="proceed"일 때 adjusted_ev_bps > 0 필수. Regime 일관성 검증 (mismatch → adverse adj > 0).
ExecutionHandoffexecution-designer → spec-writer DeviationFromBrief: |pt_pct| ≤ 0.20 AND |sl_pct| ≤ 0.20. trailing_stop=True면 activation/distance 모두 non-null.
FeedbackOutputfeedback-analyst → iter_finalize agreement_points / disagreement_points / priority_action / next_idea_seed / escape_seed 필수. Pydantic validate.
실증 사례 (iter1 smoke test, 2026-04-19): execution-designer 1st try에서 legacy flat shape 반환 → Pydantic이 18 errors로 hard-fail → 명시적 structure 예시와 함께 retry → PASS. 스키마 레이어가 정확히 설계 목적대로 동작.

Counterfactual PnL Attribution

--strict 모드에서 엔진이 위반 발생 시 개입(REJECT / FORCE SELL)하여 spec 준수 버전의 수익을 계산. clean_pnl vs bug_pnl 분리로 "진짜 edge"와 "버그가 만든 수익"을 분리.

핵심 발견 (최근 세션)

1. Pydantic Schema가 실전에서 drift를 catch

iter1 execution-designer가 legacy flat shape로 반환 → Pydantic extra="forbid"가 18 errors로 reject → retry로 정상 shape 재산출. "분석처럼 보이는 추측"을 hard-fail로 차단한 실증.

handoff schema retry loop
2. Critics가 실제 구현 버그를 발견

execution-critic이 iter1의 12 roundtrips 전부 exit_signal 태그만 붙음을 발견 → PT/SL/trailing 작동 여부 불명. 수정 후 SL 5회, time_stop 7회로 분리 확인 — critic의 진단이 실제 observability 버그 제보.

bar runner bug post-fix verified
3. Fee-Dominated Regime (반복 관찰)

KRX 21 bps, Qlib CSI500 1min 8 bps — 모두 edge가 fee에 잠식. Tick/1min horizon의 expected return (20-40 bps)이 fee × 2 (round-trip)를 못 뚫는 구조. Crypto 4 bps에서만 viable.

KRX fail CSI500 1min fail crypto viable
4. Ping-Pong을 위한 LOB 수집 시작

Bar OHLCV로는 market-making 불가능 — spread 정보가 aggregated 상태에서 lost. 2026-04-19 binance_lob_collector.py 가동 시작. 2+ 주 후 첫 LOB iter 가능. 현재 PID 1215753 수집 중.

The Tick-Level Fidelity Gap: LLM-Generated Trading Code Under Microstructure Scrutiny

타겟: NeurIPS 2026 Datasets & Benchmarks (1순위) · KDD 2027 / COLM 2027 (대안)

핵심: 기존 LLM-finance 벤치마크(FactorMiner, AlphaForgeBench 등)는 모두 bar-level 평가로 높은 pass rate를 보고하지만, tick-level + microstructure 구속 조건에서 LLM-generated code가 어떻게 실패하는지를 최초로 체계적 측정.

Contributions:

  • C1 (방법론): LLM 선언 spec으로부터의 invariant 자동 추론 — test oracle 없이 deterministic 실패 감지
  • C2 (정량화): Dual-mode (normal vs strict) counterfactual PnL attribution — 위반 크기를 PnL 단위로 분리
  • C3 (engine-agnostic): (spec, fill-list) 위에서 작동하는 측정 레이어 — multi-engine 재현
  • C4 (empirical): 4개 실패 유형 (spec drift, microstructure knowledge gap, multi-agent handoff decay, invariant taxonomy blindspot)
  • C5 (taxonomy gap): 현 7-invariant가 놓치는 microstructure-specific 위반 식별
