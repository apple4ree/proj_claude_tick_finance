프로젝트 개요

KRX 10-level 호가잔량(H0STASP0) tick 데이터 및 Upbit 크립토 LOB 데이터를 기반으로, LLM multi-agent 파이프라인이 트레이딩 전략을 자율 생성 → 백테스트 → 분석 → 학습하는 프레임워크입니다.

핵심 연구 질문: "LLM이 생성한 domain code(trading strategy)는 specification의 의도와 runtime behavior가 체계적으로 불일치하며, multi-agent critique와 automated invariant checker가 이를 얼마나 발견하는가?"

Spec-Invariant Checker

엔진이 spec.yaml 파라미터에서 runtime invariant를 자동 추론하여 매 fill 이벤트마다 체크합니다. Agent 수정 0.

Counterfactual PnL 귀속

--strict 모드에서 엔진이 위반 발생 시 개입(REJECT/FORCE SELL)하여 spec 준수 버전의 수익을 계산합니다.

핵심 발견: strat_0032의 +0.573% 수익 중 89.8%가 double-fill 버그에 의한 것. Clean PnL(spec 준수 시) = +0.058%에 불과.

핵심 발견 사항

1. KRX LOB Signal은 Fee를 이기지 못함

99만 ticks, 17개 feature × 6 horizons = 612 조합 분석 결과, 모든 signal의 Q5 수익이 fee(21 bps) 미만. 최고 signal(microprice_diff @ 1000t)의 Q5 = +3.72 bps — fee의 1/5 수준.

KRX fee 장벽 99만 ticks
2. 크립토에서도 대표본 시 Signal 유효하지 않음

BTC 9.5K ticks에서 10개 viable 보였으나, 174K ticks로 확장 시 0개 viable로 반전. 소표본 편향(small-sample bias)이 결론을 뒤집은 사례.

소표본 편향 BTC 174K ticks
3. 보고된 수익의 대부분이 Spec-Code Gap에서 발생

Counterfactual 분석 결과 strat_0032(best)의 수익 89.8%가 max_position_exceeded 버그에 의한 것. 이전 14회 iterate 루프는 버그로 인한 수익을 edge로 오인하고 있었음.

double-fill counterfactual 자동 검출
4. Signal Brief Gate가 낭비 방지

이전: 14회 iteration, ~20시간 → 결론 "edge 없음".
이후: Signal brief 1회 생성(~15분) + 1초 판단 → 같은 결론을 사전에 도출.

논문 방향

Spec-Code Semantic Gap in LLM-Generated Domain Programs

타겟: EMNLP / NeurIPS 2026 (5월 deadline)

핵심 주장: LLM이 생성한 domain code는 syntax/type check를 통과하지만 specification의 의도와 runtime behavior가 체계적으로 불일치한다. 7개 gap taxonomy + automated invariant checker + counterfactual PnL attribution으로 이를 정량화.

Contributions:

  • C1 (문제 정의): Spec-code semantic gap의 개념과 6개 유형 taxonomy
  • C2 (자동 검출): Spec-driven invariant checker — 7개 invariant 자동 추론
  • C3 (정량 영향): Counterfactual PnL attribution — clean vs bug 수익 분리
  • C4 (비교): LLM critic vs automated checker의 gap 발견율 비교
