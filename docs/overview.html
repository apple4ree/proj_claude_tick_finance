Tick Strategy Framework

KIS 10단 호가 데이터(H0STASP0)에 대해 전략을 자율적으로 생성 → 백테스트 → 학습하는 Claude Code 기반 루프.

1. 개요

이 프레임워크는 세 가지 설계 원칙을 따릅니다.

토큰 최적화

연산·iteration·파일 I/O는 전부 순수 Python(engine/) + CLI 스크립트(scripts/)가 처리합니다. Sub-agent는 결과 JSON만 읽습니다.

분업화

5개 전용 sub-agent가 이데이션 → 스펙 작성 → 코드 확장 → 백테스트 → 피드백 단계를 각자 담당합니다.

지식 누적

각 이터레이션의 교훈은 knowledge/lessons/에 Obsidian 호환 MD로 저장되고, networkx 기반 그래프로 탐색됩니다.

2. 아키텍처 및 자율 루프

[그림 1] 3-layer 아키텍처와 자율 피드백 루프

레이어 설명

  • Claude Layer — 실제 지능이 필요한 부분만. 각 sub-agent는 CLI 스크립트를 호출하고 JSON을 파싱합니다. 절대 raw 데이터를 직접 iterate하지 않습니다.
  • Engine Layer — pandas/numpy 기반 순수 Python. Claude가 개입하지 않는 전 영역이며, 토큰 비용 = 0입니다.
  • Data & Knowledge — 외부 CSV, 생성된 전략, 그리고 Obsidian 호환 MD 지식 베이스. knowledge/graph.py가 프런트매터·[[wikilink]]를 파싱해 networkx 그래프로 인덱싱합니다.

3. 엔진 파이프라인

[그림 2] engine/runner.py가 오케스트레이션하는 백테스트 파이프라인

중요 포인트

  • Taker-only: 현재 오더는 MARKET 또는 marketable LIMIT만 지원. 호가 10단을 level walk 하면서 평균체결가와 체결수량을 계산합니다.
  • Latency 모델: 전략이 tick T에서 주문을 내면, target_ts = T + submit_ms + jitter 로 큐에 적재되고, 이후 도착한 tick의 호가와 매칭됩니다 — 미래 정보 참조 없음.
  • Fee 모델: KRX 기준 — 커미션 1.5 bps (양방향) + 증권거래세 18 bps (매도만). spec.yaml의 fees:로 override 가능.
  • Resting limit은 현재 미지원. 체결판 큐 포지션 모델이 필요하며, 실제 체결 프린트 데이터 통합 시점에 추가 예정.

4. Spec DSL 문법

전략은 YAML 선언만으로 정의됩니다. 엔진이 이를 해석하며, 별도의 Python 파일은 필요 없습니다 (새 primitive가 필요할 때만 engine/signals.py 확장).

# strategies/<id>/spec.yaml
name: obi_momentum_bps_v2
description: "OBI + 단기 mid 상승 + 좁은 bps 스프레드 진입"
capital: 10000000

universe:
  symbols: ["005930", "000660"]    # 멀티 종목 동시
  dates:   ["20260313"]

fees:    { commission_bps: 1.5, tax_bps: 18.0 }
latency: { submit_ms: 5.0, jitter_ms: 1.0, seed: 42 }

signals:                            # 이름 있는 primitive 호출
  obi5:    { fn: obi,             args: { depth: 5 } }
  ret5:    { fn: mid_return_bps,  args: { lookback: 5 } }
  spr_bps: spread_bps                # 인자 없는 primitive

entry:
  when: "obi5 > 0.5 and ret5 > 1.0 and spr_bps < 15"
  size: 1

exit:
  when: "obi5 < 0.0 or holding_ticks > 80"

risk:
  max_position_per_symbol: 1

사용 가능한 primitive (engine/signals.py)

이름반환설명인자
midfloat(best_ask + best_bid) / 2—
spreadfloatbest_ask − best_bid (KRW)—
spread_bpsfloatspread / mid × 1e4 (cross-symbol 비교용)—
best_ask / best_bidfloattop of book—
obifloattop-N 잔량 imbalance ∈ [−1, 1]depth=5
micropricefloat잔량 가중 평균가—
total_imbalancefloat전체 ask/bid 잔량 imbalance—
volume_deltafloatACML_VOL 차분lookback=1
mid_return_bpsfloatmid 수익률 (bps)lookback=1
mid_changefloatmid 절대 차분 (KRW)lookback=1

entry/exit 표현식에서 쓸 수 있는 식별자

  • spec.signals에 선언한 모든 이름 (위 예시의 obi5, ret5, spr_bps)
  • 암묵 변수: position (현재 수량), holding_ticks (보유 tick 수), mid_now, spread_now
  • 리터럴/연산: 숫자, True/False (또는 true/false), and/or/not, 비교(< <= > >= == !=), 산술(+ − * / // % **), 3항(x if c else y), min/max/abs

⚠ DSL 평가기는 AST 화이트리스트입니다. 속성 접근, 속성 호출, 컴프리헨션, 람다는 모두 차단됩니다.

5. 사용법

5-1. Claude Code에서 슬래시 커맨드

Claude Code를 proj_claude_tick_finance/에서 실행하면 .claude/agents, .claude/skills, .claude/commands가 자동 로드됩니다.

# 한 이터레이션: 시드 → 전략 → 백테스트 → 레슨
/new-strategy "obi(5)가 0.5 이상이고 스프레드가 5 bps 미만일 때 long 진입"

# 완전 자율 루프 — N회, 시드는 이전 이터레이션의 next_idea_seed로 연결
/iterate 5 "좁은 스프레드 구간에서 order-flow imbalance momentum 캡처"

5-2. CLI 직접 호출 (디버깅·검증용)

# 0) 사용 가능한 날짜/종목 조회
python -m engine.data_loader list-dates
python -m engine.data_loader list-symbols --date 20260313
python -m engine.data_loader summary --date 20260313 --symbol 005930

# 1) 새 전략 디렉터리 생성
python scripts/new_strategy.py --name my_idea \
    --from strategies/_examples/obi_momentum.yaml

# 2) spec.yaml 편집 후 검증
python scripts/validate_spec.py strategies/<id>/spec.yaml

# 3) 백테스트 실행 (결과는 stdout + report.json 양쪽으로)
python -m engine.runner \
    --spec strategies/<id>/spec.yaml \
    --out  strategies/<id>/report.json \
    --summary

# 4) 레슨 기록 + 지식 그래프 업데이트
python scripts/write_lesson.py \
    --title "좁은 스프레드만으론 edge 부족" \
    --body  "Observation: ...\nWhy: ...\nHow to apply next: ..." \
    --tags  "spread,edge,failure" \
    --source strat_20260414_0001_my_idea \
    --metric "return_pct=-0.5 trades=120"
python knowledge/graph.py build

# 5) 프로젝트 상태 조회
python scripts/list_strategies.py
python scripts/search_knowledge.py --query spread --top 5
python knowledge/graph.py stats
python knowledge/graph.py related spread

9. 확장 포인트 / 백로그

Resting limit + queue model

현재 taker-only. 체결 프린트 데이터를 통합해 큐 포지션 모델을 구현하면 maker-side 전략까지 확장 가능.

qlib 엄밀성 체크리스트

walk-forward, 룩어헤드 방지, out-of-sample 검증을 metrics.py에 후처리 단계로 연결.

Multi-day rollover

현재 spec.universe.dates는 리스트로 지원되나, overnight 포지션 처리 정책이 아직 단순함. 전일 청산 / 전일 유지 옵션 필요.

심볼별 파라미터 오버라이드

현 spec은 모든 종목에 동일 threshold 적용. 종목별 정규화 임계값을 허용하면 유니버스 확장 시 유리.
