1. 개요

세 가지 설계 원칙:

Signal / Execution 분리

alpha-designer가 신호 edge(WHEN/WHY)를, execution-designer가 주문 메커닉(HOW)을 각각 담당. 진입 타이밍과 체결 전략을 독립적으로 최적화.

학습 루프

매 이터레이션마다 feedback-analyst가 WIN/LOSS 체결 시점 LOB 컨텍스트를 분석하고, Alpha Critique + Execution Critique로 구조화된 교훈을 남김.

자기 진화

meta-reviewer가 K회 마다 프레임워크 자체(engine/, DSL, knowledge 카테고리)를 감사하고 구조적 개선을 지시. 파이프라인이 고정되지 않음.

5. Parameter Optimization (Optuna)

scripts/optuna_sweep.py: spec.yaml에 optuna: 블록을 추가하면 TPE/Random sampler로 파라미터 공간을 탐색합니다.

목적: IS Edge Validation

IS 데이터에서 과적합조차 못하면 전략 구조 자체에 edge가 없다는 판정. 과적합 ≠ 최종 파라미터 — 구조 검증 도구.

주의: N ≥ 50 권장

N_roundtrips < 30에서 최적화된 threshold는 overfit. 최종 파라미터 선정은 walk-forward OOS 검증 후에.

# spec.yaml에 추가
optuna:
  n_trials: 50
  metric: return_pct       # return_pct | sharpe_annualized | composite
  min_trades: 5
  search_space:
    profit_target_bps:
      type: float
      low: 60.0
      high: 400.0
      step: 10.0
    stop_loss_bps:
      type: float
      low: 20.0
      high: 150.0
      step: 10.0
    obi_threshold:
      type: float
      low: 0.2
      high: 0.7
      step: 0.05
# 실행
python scripts/optuna_sweep.py \
    --spec strategies/<id>/spec.yaml \
    --n-trials 50 \
    --metric composite \
    --out-spec strategies/<id>/spec_best.yaml   # 최적 파라미터 저장

# agent 연동용 JSON 출력
python scripts/optuna_sweep.py --spec ... --json

8. KRX Fee & Matching Model

Fee Model

Commission 1.5 bps (BUY+SELL) + Sell tax 18.0 bps = Round-trip 19.5 bps.
Configurable per spec: fees.commission_bps, fees.tax_bps.

Latency Model

Submit delay 5ms ± 1ms jitter. Order queued at T + submit_ms + jitter, matched against future ticks. No lookahead.

Resting Limit Queue Model

Passive LIMIT: queue_ahead = LOB depth at submission. Decremented by depth drops each tick. Fill when queue_ahead ≤ 0 AND is_marketable. Fill price = limit_price (maker).

Break-even Math

WRBE = stop / (target + stop). PT=150, SL=50 → WRBE=25%. Actual WR must exceed this after 19.5 bps round-trip cost.

12. Roadmap

Walk-forward Optuna

IS 최적화 → OOS 자동 검증 파이프라인. 현재 Optuna는 IS 단독. OOS pass/fail gate 추가 필요.

Multi-day Overnight

현재 매일 EOD 강제 청산. Overnight position 유지 + gap risk 모델 옵션.

Per-symbol Parameter Override

현재 모든 종목에 동일 threshold. 종목별 OBI/spread 정규화 임계값 허용.

Execution Replay

체결 프린트 데이터 통합으로 queue 모델 정밀도 향상. 현재는 LOB depth proxy.
