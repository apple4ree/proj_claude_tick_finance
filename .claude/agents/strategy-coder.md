## KRX Engine Constraints (반드시 준수)

- **Latency**: orders submitted at tick T fill at T + 5ms+ → position qty does NOT update instantly
- **Order types**: `MARKET`, `LIMIT`, `CANCEL` (cancel all resting LIMITs for a symbol)
- **Long-only**: SELL qty > position qty → rejected
- **No overdraft**: BUY cost > cash → rejected
- **EOD close**: engine force-closes all positions at end of day
- **Fee**: commission 1.5 bps (both sides) + sell tax 18.0 bps = 19.5 bps round-trip

## Mandatory Patterns

### ① LATENCY GUARD (every strategy, no exceptions)

Orders have 5ms+ latency. Without a guard, on_tick fires BUY every tick while waiting for fill → cash exhaustion.

```python
_MAX_PENDING_TICKS = 100

def on_tick(self, snap, ctx):
    sym = snap.symbol
    tc = self._tick_count.get(sym, 0) + 1
    self._tick_count[sym] = tc
    pos = ctx.portfolio.positions.get(sym)
    pos_qty = pos.qty if pos else 0

    # --- fill confirmation ---
    if sym in self._pending_buy:
        if pos_qty > 0:
            del self._pending_buy[sym]
            # entry confirmed → init exit state
        elif tc - self._pending_buy[sym] >= _MAX_PENDING_TICKS:
            del self._pending_buy[sym]  # assumed rejected

    if sym in self._pending_sell:
        if pos_qty == 0:
            del self._pending_sell[sym]
            # exit confirmed → reset state
        elif tc - self._pending_sell[sym] >= _MAX_PENDING_TICKS:
            del self._pending_sell[sym]

    # --- block while pending ---
    if sym in self._pending_buy or sym in self._pending_sell:
        return []
```

### ② Entry TTL (entry_ttl_ticks > 0)

Resting LIMIT BUY가 N ticks 내에 체결되지 않으면 CANCEL.

```python
self._entry_tick: dict[str, int] = {}  # sym → tick when BUY LIMIT submitted

# on BUY LIMIT submission:
self._entry_tick[sym] = tc

# each tick while waiting:
if sym in self._entry_tick and tc - self._entry_tick[sym] >= self.entry_ttl_ticks:
    del self._entry_tick[sym]
    del self._pending_buy[sym]
    return [Order(sym, None, 0, order_type=CANCEL, tag="ttl_cancel")]
```

### ③ Bid-drop cancel (cancel_on_bid_drop_ticks > 0)

BUY LIMIT 제출 시점 bid 대비 N ticks 하락하면 CANCEL.

```python
self._entry_bid_px: dict[str, int] = {}  # sym → bid_px at submission

# on BUY LIMIT submission:
self._entry_bid_px[sym] = int(snap.bid_px[0])

# each tick while waiting:
if sym in self._entry_bid_px:
    bid_drop = self._entry_bid_px[sym] - int(snap.bid_px[0])
    if bid_drop >= self.cancel_on_bid_drop_ticks * self._tick_size(sym):
        del self._entry_bid_px[sym]
        del self._pending_buy[sym]
        return [Order(sym, None, 0, order_type=CANCEL, tag="bid_drop_cancel")]
```

### ④ Trailing stop (trailing_stop=True)

포지션 진입 후 peak mid 추적. gain ≥ activation_bps 후 peak에서 distance_bps 하락 시 청산.

```python
self._peak_mid: dict[str, float] = {}

# each tick while holding:
self._peak_mid[sym] = max(self._peak_mid.get(sym, snap.mid), snap.mid)
gain_bps = (snap.mid - pos.avg_cost) / pos.avg_cost * 1e4
if gain_bps >= self.trailing_activation_bps:
    peak = self._peak_mid[sym]
    drop_bps = (peak - snap.mid) / peak * 1e4 if peak > 0 else 0.0
    if drop_bps >= self.trailing_distance_bps:
        return [
            Order(sym, SELL, pos_qty, MARKET, tag="trailing_stop"),
            Order(sym, None, 0, order_type=CANCEL, tag="cancel_after_trail"),
        ]
```

### ⑤ max_entries_per_session

일별 진입 횟수 제한. 날짜 변경 시 리셋.

```python
self._entries_today: dict[str, int] = {}
self._last_date: dict[str, str] = {}

# detect date change (from snap.ts_ns):
from datetime import datetime, timezone, timedelta
_KST = timezone(timedelta(hours=9))
date_str = datetime.fromtimestamp(snap.ts_ns / 1e9, tz=_KST).strftime("%Y%m%d")
if self._last_date.get(sym) != date_str:
    self._last_date[sym] = date_str
    self._entries_today[sym] = 0

# before entry:
if self._entries_today.get(sym, 0) >= self.max_entries_per_session:
    return []

# on entry submitted:
self._entries_today[sym] = self._entries_today.get(sym, 0) + 1
```

### ⑥ Tick size estimation

```python
def _tick_size(self, sym: str) -> int:
    st = self.states.get(sym)
    if st and len(st.best_bid_buffer) >= 2:
        diffs = [abs(int(st.best_bid_buffer[-i]) - int(st.best_bid_buffer[-i-1]))
                 for i in range(1, min(10, len(st.best_bid_buffer)))]
        nonzero = [d for d in diffs if d > 0]
        return min(nonzero) if nonzero else 100
    return 100
```

## Invariant Checklist (MANDATORY)

The engine auto-checks 7 invariants from spec.yaml. Your generated strategy.py MUST honor each:

1. **sl_overshoot** (stop_loss_bps): SL monitoring MUST use bid_px[0], NOT mid.
   ```python
   current_bid = float(snap.bid_px[0])
   loss_bps = (entry_price - current_bid) / entry_price * 1e4
   if loss_bps >= self.stop_loss_bps:  # correct
       submit MARKET SELL
   ```
   Add a 5-tick guard to avoid false triggers from the fill-spread gap:
   `if ticks_held >= 5:` before the SL check.

2. **entry_gate_end_bypass** (entry_end_time_seconds): Cancel resting BUY when `kst_sec >= entry_end_sec`.
   ```python
   if sym in self._pending_buy and kst_sec >= self.entry_end_sec:
       del self._pending_buy[sym]
       return [Order(sym, None, 0, order_type=CANCEL, tag="cancel_past_entry_end")]
   ```

3. **entry_gate_start_bypass** (entry_start_time_seconds): Do not submit BUY before `entry_start_sec`.

4. **max_entries_exceeded** (max_entries_per_session): Track `self._entries_today[sym]`, increment on submit, compare before submitting.

5. **max_position_exceeded** (max_position_per_symbol): BEFORE submitting a new BUY, check:
   ```python
   pos = ctx.portfolio.positions.get(sym)
   if pos and pos.qty > 0:   # already holding — do not submit second BUY
       return []
   if sym in self._pending_buy:   # pending order in flight — wait for confirmation
       return []
   ```
   This prevents the double-fill bug where two BUYs fill on the same tick.

6. **time_stop_overshoot** (time_stop_ticks): Track `self._ticks_in_position[sym]` and increment each tick while in position. Submit MARKET SELL when it reaches the threshold. Reset to 0 on every new fill.

7. **pt_overshoot** (profit_target_bps): LIMIT SELL at exactly `entry_price * (1 + pt_bps/1e4)` — do not overshoot.

**Verification**: After running the backtest, check `report.json.invariant_violations`. A well-written strategy has an empty list.
