# IS / OOS 데이터 분할 정의

## 기준

| 기간 | 날짜 | 일수 | 용도 |
|------|------|------|------|
| In-Sample (IS) | 20260316, 20260317, 20260318, 20260319, 20260320, 20260323, 20260324, 20260325 | 7일 | 전략 개발, 파라미터 탐색, 백테스트 |
| Out-of-Sample (OOS) | 20260326, 20260327, 20260330 | 3일 | 최종 검증 전용 — 전략 개발 중 절대 사용 금지 |

## 유니버스

- IS/OOS 모두 40개 심볼 사용 가능
- 기준 확인: `python -m engine.data_loader list-symbols --date 20260316`
- 기본 개발 대상: `005930`, `000660` (충분한 tick 수 보장)

## 사용 규칙

1. `spec.yaml`의 `universe.dates`는 **IS 날짜만** 포함한다
2. OOS(20260326/20260327/20260330)는 `/iterate` 루프 내에서 **절대 사용하지 않는다**
3. OOS 검증은 별도 명시적 요청이 있을 때만, IS 전략 개발 완료 후 수행한다
4. IS 내에서 calibration check(Step 2d)는 20260316 기준으로 실행한다

## 배경

- 20260313(기존 단일 테스트일)은 극단적 bid-heavy 하루(median imbalance=+0.467)로 대표성 없음
- IS 7일은 서로 다른 tick 수(71k~112k)를 보여 다양한 regime을 포함
- 단일 날짜 백테스트는 in-sample/out-of-sample 분리 불가 → 유효한 edge 검증 불가능
